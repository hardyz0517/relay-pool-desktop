# Sub2API 界面设计报告

## 1. 核心气质

Sub2API 的界面是明显偏工具型的桌面管理壳：浅色主题、紧凑密度、左侧导航、强边界内容面板。它不追求营销感，而是优先服务扫描、比较和高频切换。

## 2. 侧边栏逻辑

它的侧边栏非常稳定。展开态和收起态保留同一套 DOM，收起时只收缩宽度和文字空间，不会删除内容树，因此不会出现布局漂移。

关键做法是：
- 侧边栏宽度只分两档
- 导航项始终复用同一结构
- 图标始终可见
- 文本通过 `max-width`、`opacity`、`transform` 收缩
- 底部控制区也复用同一套行语法

## 3. 信息层级

Sub2API 用小标题、低对比度分隔线、紧凑卡片来组织层级。每个区块只做一件事，视觉节奏也很统一。

## 4. 交互模型

它的交互很克制：切换、收起、状态徽标、路由导航、密度较高的表格面板。大多数交互都是局部状态变化，而不是弹窗或向导式流程。

## 5. 它为什么显得成熟

- 侧边栏不会在不同状态下用不同方式重新排版
- 标签是淡出或裁切，不是通过重建 DOM 消失
- 宽度、内边距、间距是成体系的
- 全局布局有稳定的壳，边界清楚
- UI 始终偏向操作清晰，而不是视觉新鲜感

## 6. 对 Relay Pool Desktop 的启发

- 所有侧边栏行尽量复用同一套结构
- 标签收起只做视觉收缩，不做结构重写
- Header、Nav、Footer、Button 的 padding 和宽度步进要统一
- Footer 在收起态也要保留状态，不要突然改几何
- 壳层要当成稳定框架，不要塞太多特例
