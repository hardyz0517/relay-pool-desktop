# 首页最近使用空状态设计

## 目标

首页“最近使用”在工作台数据加载完成且没有请求记录时，不再留下空白内容区，而是显示明确的空状态。

## 界面

- 保留现有“最近使用”区块标题和容器结构。
- 空状态使用低饱和灰色图标底板、收件箱图标、主文案“暂无使用记录”和说明“开始使用 API 后，您的使用历史将显示在这里。”。
- 布局在内容区水平、垂直居中，并提供稳定的最小高度，保持桌面工具的紧凑浅色风格。
- 有请求记录时继续显示现有最多五条记录，行样式和字段不变。

## 状态与数据流

- 为工作台初次加载增加显式完成状态。
- 初次加载尚未结束时不渲染“暂无使用记录”，避免空状态闪烁。
- `loadDashboardWorkspace()` 成功返回后标记加载完成；此时 `requestLogs.length === 0` 才渲染空状态。
- 加载失败继续沿用现有 toast 错误提示，不把失败误报成“暂无使用记录”。

## 实现边界

- 仅修改首页组件和对应的首页布局回归脚本。
- 不改变请求日志查询、排序、条数限制或后端数据模型。
- 不重构共享 `EmptyState`，避免影响其他页面；首页按目标截图组合现有 Lucide 图标和局部样式。

## 验证

- 回归脚本断言空状态受“加载完成且请求日志为空”条件控制，并包含目标文案和图标。
- 运行该回归脚本，先确认修改生产代码前会失败，再确认实现后通过。
- 运行 TypeScript/Vite 构建，确认前端类型和打包无回归。
