# 路由状态页精简与对齐设计

## 目标

精简本地路由状态页，删除“排障入口”和“最近一次路由解释”，并修复“当前主 Key”卡片与“本地端点”卡片顶部不对齐的问题。

## 范围

- 从 `LocalRoutingStatusTab` 删除排障入口区块和路由解释区块。
- 清理仅用于排障入口的 `onOpenPage` 属性传递、页面类型、图标和按钮引用。
- 删除不再被使用的 `RouteExplanationPanel` 组件，但保留后端路由决策采集、工作区查询和数据类型。
- 两列概览网格使用顶部对齐，左右卡片标题和内容容器保持同一基线。

## 非目标

- 不修改路由选择、候选排序或故障切换逻辑。
- 不删除请求日志中的路由决策数据。
- 不改变状态页和编辑页之间的切换方式。

## 交互与布局

状态页首屏保留“本地端点”“当前主 Key”和“候选顺位”。两列概览容器增加顶部对齐约束，使内容高度不同的卡片不再通过网格拉伸改变标题轨道高度。移动端继续按单列自然排列。

## 验证

- 回归脚本确认页面源码不再包含两个被删除区块及其导航属性链路。
- 回归脚本确认两列概览网格包含顶部对齐约束。
- 运行 TypeScript 与 Vite 生产构建。
