# Relay Pool Desktop 界面审计报告

## 1. 壳层问题

当前壳层里，侧边栏同时混用了多套布局语法。部分行用 `grid`，部分行用 `flex`，部分状态通过隐藏节点处理，部分状态通过保留节点处理。这种不一致是侧边栏之前不稳定的主要原因。

壳层里还塞了太多状态特例。折叠逻辑、底部状态、标签、按钮都直接写在一个组件里，小改动很容易牵动一串排版变化。

## 2. 侧边栏问题

- 收起宽度之前是靠感觉反复调的，没有统一的行系统
- Footer 内容在不同状态下曾改变高度
- 标签曾以会影响布局时序的方式被隐藏
- 折叠按钮和导航项没有共享同一套间距语法
- 侧边栏整体已经接近桌面工具，但状态切换成熟度明显不够

## 3. 页面层问题

### 总览

总览页把实时代理状态、假数据摘要卡、说明性面板混在一屏里。信息是有的，但层级还偏急，什么都想一次讲完。

### 设置

设置页已经把代理控制和路由偏好分开了，但表单还是偏密，一些控件仍只是普通字段，缺少更明确的交互表达。

### 日志

日志页结构比其他页更好，但右侧 inspector 和左侧表格仍像两个并排区域，不算一个完整的分析流程。

## 4. 交互成熟度缺口

- 状态切换不总是锚定在固定几何上
- 某些标签和状态文本在组件级别被过度处理
- 壳层还需要更清晰的可复用行契约
- 一些页面的信息密度高，但视觉秩序不够
- 主要操作和状态信号没有完全统一在同一套布局语言里

## 5. 下一步建议

- 统一侧边栏行宽和折叠行为
- 如果别处也在用同类模式，抽一个更紧凑的 `shell` 行原语
- 收紧 Footer 行为，避免状态切换改变高度
- 把总览和日志的卡片/面板节奏再理顺
- 少用临时性的宽度微调，多用 token 化的间距
